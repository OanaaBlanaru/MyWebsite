Environment Variables

An environment variable is a user-definable value that can affect the way running processes will behave on a computer. Environment variables are part of the environment in which a process runs. For example, a running process can query the value of the TEMP environment variable to discover a suitable location to store temporary files, or the HOME or USERPROFILE variable to find the directory structure owned by the user running the process.

Types:

  • System variables
  • User variables
  • Path variable

Important: Environment variables are case sensitive!

Setup & Configuration

Setup Environment Variables For JDK 11

  1. Go to Environment Variables. In the section System Variables click New
  2. Add veriable path and save. Name it "JAVA_HOME"
  3. Add new variable bin to PATH [click on PATH variable > click New and save]. Use this text: "%JAVA_HOME%\bin"
  4. Test in command prompt using "java -version"

Setup Environment Variables For Maven

  1. Go to Environment Variables. In the section System Variables click New
  2. Add variable and save. Name it "MAVEN_HOME"
  3. Add new variable bin to PATH [click on PATH variable > click New and save]. Use this text: "%MAVEN_HOME%\bin"
  4. Test in command prompt using "mvn --version"

Setup Environment Variables For Intellij

  1. Go to Environment Variables and verify is the path is already there
  2. If not, in the section System Variables click New. Add variable and save it. Name it "IntelliJ IDEA Community Edition"
  3. Add new variable bin to PATH [click on PATH variable > click New and save]. Use this text: "%IntelliJ IDEA Community Edition%"

Configure GIT

git-scm.com

  1. Create a new user environment variable called HOME pointing to your user's home directory (C:\Users\username)
  2. Add the GIT install path to PATH Environment variable
  3. Test the GIT installation with command "git"
  4. Configure GIT with commands:
    • git config --global user.name "your user"
    • git config --global user.email "your email"
